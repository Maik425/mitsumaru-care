# PR-004: 実装ノート

## UI変更を避けるための原則

- 画面構造、見出し文言、ボタン文言は既存維持（E2Eが依存）
- strict mode違反（重複見出し/ボタン）は引き続き回避（`aria-hidden`や条件レンダリング）
- セレクタは `data-testid` を最優先し、必要最小限のみ追加

## オフライン/エラー処理

- SWが`/admin/shifts`をハンドリングしているため、UI側も同一文言を維持
- 例外は共通関数でDOMメッセージ（既存CSS）を生成し、文言は既存に合わせる

## API接続の段階的切り替え

- フェーズ1: UIからtRPC呼び出し→スタブ応答→E2E維持
- フェーズ2: Prisma接続の実データ（別PRで）
- UIから見た差分が出ないようにadapter関数で吸収

## ロール/ガード

- `localStorage`モックを当面継続
- 役割チェックは`src/lib/auth`のヘルパーに集約して重複排除

## テスト

- URLは正規表現での一致を維持
- `getByRole`, `getByLabel`, `getByTestId`の優先順位を明確にして重複防止
