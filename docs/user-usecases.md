# ユーザー別ユースケース定義書

## 📋 概要

このドキュメントでは、みつまるケアシステムの各ユーザーが実際に使用するであろうユースケースを詳細に定義します。これらのユースケースに基づいて、E2Eテストを作成し、システムの品質を保証します。

## 👥 ユーザー分類

### 1. 管理者（ADMIN/OWNER）

- **施設長** - 全機能へのアクセス権限
- **主任** - シフト管理・勤怠管理・職員管理の権限

### 2. 一般職員（MEMBER）

- **介護福祉士** - 勤怠管理・休暇申請の権限
- **看護師** - 勤怠管理・休暇申請の権限
- **その他職員** - 勤怠管理・休暇申請の権限

## 🎯 管理者ユースケース

### UC-ADM-001: 月間シフト作成

#### 概要

管理者が一ヶ月分の勤務表を自動で作成する

#### 前提条件

- 管理者としてログインしている
- シフト形態・職員情報・配置ルールが登録済み

#### 基本フロー

1. 管理者ダッシュボードにアクセス
2. 「シフト作成」メニューを選択
3. 対象月を選択（例：2025年3月）
4. シフト形態を確認（早番・日勤・遅番・夜勤）
5. 職員情報を確認（人数・スキル・希望休）
6. 配置ルールを確認（一日の必要人数）
7. 「自動シフト生成」ボタンをクリック
8. 生成されたシフトを確認
9. 必要に応じて手動調整
10. 「シフト確定」ボタンをクリック
11. 職員への通知を確認

#### 代替フロー

- **A1**: シフト生成エラー時
  1. エラーメッセージを確認
  2. 入力情報を修正
  3. 基本フロー6から再実行

- **A2**: 手動調整が必要な場合
  1. 該当セルをクリック
  2. シフト形態を変更
  3. 職員を変更
  4. 基本フロー9から再実行

#### 事後条件

- 月間シフトが確定される
- 職員にシフト情報が通知される
- シフトデータがデータベースに保存される

### UC-ADM-002: シフト編集・調整

#### 概要

確定済みのシフトを編集・調整する

#### 前提条件

- 管理者としてログインしている
- 編集対象のシフトが存在する

#### 基本フロー

1. シフト管理画面にアクセス
2. 編集対象のシフトを選択
3. シフト詳細画面を表示
4. 編集したいセルをクリック
5. シフト形態・職員・時間を変更
6. 変更内容を保存
7. 変更内容を確認
8. 必要に応じて承認者に承認依頼

#### 代替フロー

- **A1**: 承認が必要な変更の場合
  1. 承認者を選択
  2. 変更理由を記入
  3. 承認依頼を送信
  4. 承認待ち状態になる

#### 事後条件

- シフトが更新される
- 変更履歴が記録される
- 関係者に変更通知が送信される

### UC-ADM-003: シフト交換申請の承認・却下

#### 概要

職員間のシフト交換申請を承認または却下する

#### 前提条件

- 管理者としてログインしている
- シフト交換申請が存在する

#### 基本フロー

1. シフト交換管理画面にアクセス
2. 承認待ちの申請一覧を確認
3. 対象申請を選択
4. 申請内容を詳細確認
5. 承認・却下を選択
6. 必要に応じてコメントを記入
7. 決定を確定

#### 代替フロー

- **A1**: 却下する場合
  1. 却下理由を記入
  2. 申請者に却下通知を送信

#### 事後条件

- 申請が承認・却下される
- 申請者に結果が通知される
- 承認された場合はシフトが更新される

### UC-ADM-004: 役割表作成

#### 概要

シフトで決まった勤務者に対して、一日の業務担当を割り当てる

#### 前提条件

- 管理者としてログインしている
- 対象日のシフトが確定済み

#### 基本フロー

1. 役割表作成画面にアクセス
2. 対象日を選択
3. 勤務メンバーを確認
4. 役割テンプレートを適用
5. 自動割り当てを実行
6. 割り当て結果を確認
7. 必要に応じて手動調整
8. 役割表を確定

#### 代替フロー

- **A1**: 自動割り当てが不完全な場合
  1. 不足している役割を手動で割り当て
  2. スキル条件を満たす職員を選択
  3. 基本フロー7から再実行

#### 事後条件

- 役割表が確定される
- 各職員に担当業務が割り当てられる
- 役割表データがデータベースに保存される

### UC-ADM-005: 勤怠状況の確認・承認

#### 概要

職員の勤怠状況を確認し、修正申請を承認する

#### 前提条件

- 管理者としてログインしている
- 勤怠データが存在する

#### 基本フロー

1. 勤怠管理画面にアクセス
2. 対象期間・職員を選択
3. 勤怠状況を確認
4. 異常（遅刻・早退・残業）をチェック
5. 修正申請がある場合は内容を確認
6. 申請を承認または却下
7. 必要に応じてコメントを記入

#### 代替フロー

- **A1**: 異常が発見された場合
  1. 該当職員に連絡
  2. 事情を確認
  3. 必要に応じて勤怠を修正

#### 事後条件

- 勤怠状況が把握される
- 修正申請が処理される
- 異常が記録される

### UC-ADM-006: 職員管理

#### 概要

職員の登録・編集・削除を行う

#### 前提条件

- 管理者としてログインしている

#### 基本フロー

2. 職員一覧を確認
3. 新規職員登録・既存職員編集・職員削除を選択
4. 必要情報を入力・編集
5. 保存を実行

#### 事後条件

- 職員情報が更新される
- 職員データがデータベースに保存される

#### 基本フロー

1. 各種マスターデータ管理画面にアクセス
2. データ一覧を確認
3. 新規登録・編集・削除を選択
4. 必要情報を入力・編集
5. 保存を実行

#### 事後条件

- マスターデータが更新される
- データがデータベースに保存される

### UC-ADM-008: 休暇申請の承認・却下

#### 概要

職員の休暇申請を承認または却下する

#### 前提条件

- 管理者としてログインしている
- 休暇申請が存在する

#### 基本フロー

1. 休暇管理画面にアクセス
2. 承認待ちの申請一覧を確認
3. 対象申請を選択
4. 申請内容を詳細確認
5. 承認・却下・条件付き承認を選択
6. 必要に応じてコメントを記入
7. 決定を確定

#### 代替フロー

- **A1**: 却下する場合
  1. 却下理由を記入
  2. 申請者に却下通知を送信

- **A2**: 条件付き承認する場合
  1. 承認条件を記入
  2. 条件付き承認を実行

#### 事後条件

- 申請が承認・却下される
- 申請者に結果が通知される
- 承認された場合は休暇が記録される

### UC-ADM-009: レポート・エクスポート機能

#### 概要

勤怠・シフト・休暇の各種レポートを生成し、データをエクスポートする

#### 前提条件

- 管理者としてログインしている

#### 基本フロー

2. レポート種別を選択（日次・月次・個人別）
3. 対象期間・対象者を設定
4. レポートを生成
5. 必要に応じてExcel・PDF・CSVでエクスポート

#### 事後条件

- レポートが生成される
- データがエクスポートされる

### UC-ADM-010: 通知・コミュニケーション管理

#### 概要

システム内での通知送信とコミュニケーション機能を管理する

#### 前提条件

- 管理者としてログインしている

#### 基本フロー

1. 通知管理画面にアクセス
2. 通知テンプレートの作成・編集
3. 配信設定の管理
4. システムメッセージの作成・配信

#### 事後条件

- 通知設定が更新される
- システムメッセージが配信される

## 👤 一般職員ユースケース

### UC-EMP-001: 出退勤打刻

#### 概要

職員が実際の出勤・退勤時刻を記録する

#### 前提条件

- 一般職員としてログインしている
- 勤務予定がある

#### 基本フロー

1. 勤怠管理画面にアクセス
2. 現在の日時を確認
3. 「出勤」または「退勤」ボタンをクリック
4. 打刻時刻を確認
5. 必要に応じてコメントを記入
6. 打刻を確定

#### 代替フロー

- **A1**: 打刻時刻が予定と異なる場合
  1. 理由を記入
  2. 修正申請を送信

#### 事後条件

- 出退勤時刻が記録される
- 勤怠データがデータベースに保存される

### UC-EMP-002: 勤怠修正申請

#### 概要

勤怠データに誤りがある場合、修正を申請する

#### 前提条件

- 一般職員としてログインしている
- 修正が必要な勤怠データが存在する

#### 基本フロー

1. 勤怠管理画面にアクセス
2. 修正したい勤怠データを選択
3. 「修正申請」ボタンをクリック
4. 修正内容を入力
5. 修正理由を記入
6. 申請を送信

#### 事後条件

- 修正申請が送信される
- 管理者に申請が通知される
- 申請状態が「承認待ち」になる

### UC-EMP-003: 休暇申請

#### 概要

希望休・有給休暇の申請を行う

#### 前提条件

- 一般職員としてログインしている

#### 基本フロー

1. 休暇管理画面にアクセス
2. 「休暇申請」ボタンをクリック
3. 休暇種別を選択（希望休・有給休暇・その他）
4. 開始日・終了日を選択
5. 申請理由を記入
6. 申請を送信

#### 代替フロー

- **A1**: 有給休暇の残日数が不足している場合
  1. 残日数を確認
  2. 申請日数を調整
  3. 基本フロー4から再実行

#### 事後条件

- 休暇申請が送信される
- 管理者に申請が通知される
- 申請状態が「承認待ち」になる

4. 交換希望日時を指定
5. 交換理由を記入

#### 代替フロー

- シフト交換申請が送信される
- 交換相手に申請が通知される
- 管理者に申請が通知される

## 🔄 共通ユースケース

### UC-COM-001: ログイン・ログアウト

#### 概要

システムにログイン・ログアウトする

#### 前提条件

- 有効なアカウントが存在する

#### 基本フロー

1. ログイン画面にアクセス
2. メールアドレス・パスワードを入力
3. 「ログイン」ボタンをクリック
4. 認証が成功した場合、適切なダッシュボードに遷移

#### 代替フロー

- **A1**: 認証が失敗した場合
  1. エラーメッセージを確認
  2. 入力情報を修正
  3. 基本フロー2から再実行

#### 事後条件

- ログインが成功する
- 適切な権限でシステムにアクセスできる

### UC-COM-002: パスワード変更

#### 概要

パスワードを変更する

#### 前提条件

- ログインしている

#### 基本フロー

1. 設定画面にアクセス
2. 「パスワード変更」を選択
3. 現在のパスワードを入力
4. 新しいパスワードを入力
5. 新しいパスワードを再入力
6. 「変更」ボタンをクリック

#### 代替フロー

- **A1**: 現在のパスワードが間違っている場合
  1. エラーメッセージを確認
  2. 現在のパスワードを修正
  3. 基本フロー3から再実行

#### 事後条件

- パスワードが変更される
- 新しいパスワードでログインできる

### UC-COM-003: プロフィール管理

#### 概要

ユーザープロフィール情報を管理する

#### 前提条件

- ログインしている

#### 基本フロー

1. プロフィール画面にアクセス
2. 基本情報を確認・編集
3. 連絡先情報を更新
4. 変更を保存

#### 事後条件

- プロフィール情報が更新される

## 🧪 テスト対象ユースケース

E2Eテストでは、以下のユースケースを優先的にテストします：

1. **認証フロー** - UC-COM-001
2. **シフト管理フロー** - UC-ADM-001, UC-ADM-002
3. **勤怠管理フロー** - UC-EMP-001, UC-ADM-005
4. **役割表管理フロー** - UC-ADM-004
5. **シフト交換フロー** - UC-EMP-004, UC-ADM-003
6. **休暇管理フロー** - UC-EMP-003, UC-ADM-008
7. **マスターデータ管理フロー** - UC-ADM-006, UC-ADM-007
8. **レポート・エクスポートフロー** - UC-ADM-009
9. **通知・コミュニケーションフロー** - UC-ADM-010, UC-EMP-007

これらのユースケースは、システムの核心機能を網羅しており、ユーザーが日常的に使用する主要な機能です。
