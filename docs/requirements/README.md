# 要件定義書

## 📋 1. 全体導入

### 1.1 プロジェクト概要

**みつまるケア**は、介護施設の業務効率化を目的とした包括的なシフト・役割・勤怠管理システムです。

### 1.2 システムの特徴

- **3つの主要機能**: 月間シフト作成、役割表作成、勤怠管理
- **独立した機能**: 各機能は独立して動作
- **情報共有**: 登録情報を共有して連携する仕組み
- **一貫性**: 施設運営に必要なシフト・役割・勤怠を一貫して管理

## 🎯 2. 機能要件

### 2.1 認証・認可機能

#### 2.1.1 ログイン機能

- **要件**: 職員ごとのアカウントでログイン
- **対象**: 全職員（管理者・一般職）
- **認証方式**: メールアドレス・パスワード認証

#### 2.1.2 権限管理

- **管理者**: 全機能へのアクセス権限
- **一般職**: 勤怠管理・休暇管理のみアクセス権限

### 2.2 ダッシュボード機能

#### 2.2.1 管理者ダッシュボード

- **シフト作成**: 月間シフト作成・編集・管理
- **役割表作成**: 業務担当者の割り当て・管理
- **勤怠確認**: 職員の勤怠状況確認・承認
- **各種登録管理**: マスターデータの管理

#### 2.2.2 一般職ダッシュボード

- **勤怠管理**: 出退勤の打刻・申請
- **休暇管理**: 希望休・有給休暇の申請・管理

## 🔄 3. 主要機能詳細

### 3.1 機能①：月間シフト作成

#### 3.1.1 概要

一ヶ月分の勤務表を自動で作成する機能

#### 3.1.2 入力情報

- **シフト形態**: 早番・日勤・遅番などの勤務パターン
- **役職情報**: 介護福祉士・看護師等の職種
- **技能情報**: 食事介助・排泄介助・移乗介助等のスキル
- **配置ルール**: 一日の必要人数のテンプレート
- **休暇情報**: 希望休・交換休・有給休暇

#### 3.1.3 処理フロー

1. 各種登録情報の確認
2. シフト簡単作成（自動割り当て）
3. シフト詳細設定（最終調整）

### 3.2 機能②：役割表作成

#### 3.2.1 概要

シフトで決まった勤務者に対して、一日の業務担当を割り当てる機能

#### 3.2.2 入力情報

- **役割表テンプレート**: 業務ごとの必要人数・スキル条件
- **勤務メンバー**: シフトで決まった当日の勤務者

#### 3.2.3 処理フロー

1. 役割表テンプレートの適用
2. 勤務メンバーとの突き合わせ
3. 自動割り当ての実行
4. 必要に応じた手動調整

### 3.3 機能③：勤怠管理

#### 3.3.1 概要

シフトで作られた予定と、実際の出退勤を突き合わせる機能

#### 3.3.2 機能詳細

- **出退勤打刻**: 一般職による出勤・退勤の記録
- **予実照合**: 予定と実績の比較・確認
- **アラート機能**: 遅刻・早退・残業の通知
- **申請・承認**: 修正申請の処理・承認
- **集計・エクスポート**: 日次・月次データの出力

## 🔗 4. 機能間の連携

### 4.1 データフロー

```
各種登録管理 → シフト作成 → 役割表作成 → 勤怠管理
     ↓              ↓           ↓           ↓
  マスターデータ → 勤務予定 → 業務担当 → 実績管理
```

### 4.2 連携ポイント

- **シフト作成**: 各種登録情報を参照して自動生成
- **役割表作成**: シフトの勤務者情報を参照
- **勤怠管理**: シフトの予定と実績を照合

## 📊 5. 非機能要件

### 5.1 性能要件

- **レスポンス時間**: 画面遷移3秒以内
- **同時接続数**: 100ユーザー以上
- **データ処理**: 月間シフト生成5分以内

### 5.2 可用性要件

- **稼働率**: 99.5%以上
- **保守時間**: 月1回、深夜2時間以内

### 5.3 セキュリティ要件

- **認証**: セッション管理による認証
- **認可**: ロールベースアクセス制御
- **データ保護**: 個人情報の暗号化・アクセスログ

### 5.4 運用要件

- **バックアップ**: 日次バックアップ
- **監視**: システム稼働状況の監視
- **ログ**: 操作ログ・エラーログの記録

## 📝 6. 制約事項

### 6.1 技術制約

- **フレームワーク**: Next.js 14 (App Router)
- **データベース**: Supabase (PostgreSQL)
- **認証**: Supabase Auth
- **デプロイ**: Vercel

### 6.2 業務制約

- **対応施設**: 介護施設向け
- **利用者**: 職員（管理者・一般職）
- **業務時間**: 24時間対応（夜間は自動処理）
