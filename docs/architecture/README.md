# システム構成設計書

## 📋 1. 概要

このドキュメントでは、みつまるケアシステムの全体アーキテクチャとシステム構成について説明します。

### 1.1 設計方針

- **レイヤードアーキテクチャ**による関心の分離
- **ドメイン駆動設計（DDD）**の原則に従った構造
- **スケーラビリティ**を考慮した設計
- **保守性**と**拡張性**の両立

### 1.2 アーキテクチャパターン

- **フロントエンド**: Next.js App Router + React
- **バックエンド**: tRPC + DDD Architecture
- **データベース**: Supabase (PostgreSQL)
- **認証**: Supabase Auth

## 🔐 4. 認証・認可アーキテクチャ

### 4.1 認証フロー

```
1. ユーザーログイン → Supabase Auth
2. JWTトークン発行 → フロントエンド保存
3. API呼び出し → tRPCコンテキストで認証
4. 権限チェック → ロールベースアクセス制御
```

### 4.2 権限管理

```typescript
enum UserRole {
  OWNER = 'OWNER', // 全権限
  ADMIN = 'ADMIN', // 管理権限
  MEMBER = 'MEMBER', // 一般権限
}

// 権限チェック例
const canManageShifts = (userRole: UserRole) => {
  return userRole === UserRole.OWNER || userRole === UserRole.ADMIN;
};
```

## 🔄 6. データフロー

### 6.1 シフト作成フロー

```
1. フロントエンド → シフト作成フォーム入力
2. tRPC → シフト作成API呼び出し
3. アプリケーション層 → シフト作成ユースケース実行
4. ドメイン層 → シフトエンティティ作成・検証
5. インフラ層 → データベース保存
6. レスポンス → フロントエンド更新
```

### 6.2 勤怠管理フロー

```
1. 一般職 → 出退勤打刻
2. システム → 予定と実績照合
3. 管理者 → 勤怠状況確認・承認
4. システム → 勤怠データ確定
5. レポート → 集計・エクスポート
```
